# Contributing to TEC

Thank you for your interest in contributing to TEC. We welcome
contributions from the community to help improve the project. Whether
you're reporting a bug, suggesting a feature, or submitting code
changes, your input is valuable.

Please take a moment to review this document to make the contribution
process easy and effective for everyone involved.

## Code of Conduct

This project adheres to the [Contributor Covenant Code of
Conduct](https://www.contributor-covenant.org/version/2/0/code_of_conduct.html).
By participating, you are expected to uphold this code. Please report
unacceptable behavior to [TEC Discussions](https://github.com/olddeuteronomy/tec/discussions/).

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub with the following details:
- A clear, descriptive title.
    - Steps to reproduce the bug.
- Expected and actual behavior.
    - Screenshots or logs if applicable.
- Your environment (OS, compiler version, etc.).

Use the "Bug Report" issue template if available.

### Suggesting Enhancements

For feature requests or improvements:
- Open an issue with a descriptive title.
    - Explain the problem you're solving or the enhancement's value.
- Provide any relevant examples or mockups.
    - Use the "Feature Request" template if provided.

### Pull Requests

We love pull requests! Follow these steps:
1. Fork the repository and create a new branch from `main` (e.g., `feature/my-new-feature` or `fix/my-bug-fix`).
2. Make your changes, ensuring they follow the project's coding style.
3. Add or update tests if applicable.
4. Commit your changes with clear, descriptive messages.
5. Push to your fork and submit a pull request to the `main` branch.
6. In the PR description, reference any related issues (e.g., "Closes #123").

Your PR will be reviewed as soon as possible. If changes are needed,
we'll provide feedback.

#### First-Time Contributors
Look for issues labeled "good first issue" or "help wanted" to get started.

## Development Setup

To set up the project locally:
1. Clone the repository: `git clone https://github.com/olddeuteronomy/tec.git`
2. No external dependencies required (Zlib and gRPC are optional).

## Style Guidelines

- Follow TEC source code style for code formatting.
    - Use meaningful variable names and comments.
- Ensure code is linted and passes all tests before submitting.
