Detailed Description

Abstract interface for a server implementation.

Defines the minimum set of methods for a server, including starting and shutting down operations, with associated signals for state transitions. Derived classes must implement these methods to provide specific server functionality.

virtual void tec::Server::start ( Signal &  sig_started,
Status &  status 
)
pure virtual

Starts the server.

Initiates the server’s operation and signals completion via sig_started. If successful, the server may run indefinitely (e.g., in gRPC) until shutdown() is called from another thread. In case of timeout or error, the status argument is updated with the appropriate error code (e.g., Error::Kind::Timeout).

Parameters
sig_startedSignal set when the server has started (possibly with an error).
statusUpdated with the result of the operation (e.g., Error::Kind::Ok or Error::Kind::Timeout).
Note
In some implementations (e.g., gRPC), this method may not return until shutdown() is called.
